某se情直播App Api接口加解密分析

发表于 2021-04-16 | 分类于 spider

背景

在移动互联网时代，se情行业自然没有落下直播这个风口，为了防止被执法，api服务器往往部署在国外，但是为了提升用户体验，静态资源、直播视频流等往往选择接入国内云厂商以提升访问速度。
当然云厂商可以直接对资源进行审核，但有时候资源是加密的，而且为了证据链完整，从app的源头上分析就显得很有必要了。
逆向分析这些非法的app接口，提取出其使用到的服务及域名，保存事例资源，为安全及内容审核团队提供决策依据。

前置条件

  • root安卓手机(magisk)
  • 安装xposed框架
  • 安装CryptoFucker自吐加解密算法插件
  • 安装Python加解密库pip install pycryptodome

CryptoFucker 简介

CryptoFucker为xposed下一款自吐加解密算法插件，其hook住了Java sdk的消息摘要算法、对称加密算法、非对称加密算法。
在安卓下，App只要调用了Java sdk下的各种算法的API，都可以将对应的加密数据、解密数据、以及密钥输出到/sdcard/ydsec/{pkg}.txt

抓包分析

直播房间列表http请求recommendList

  • 使用了http 2.0，也可以降级到http1.1
  • 使用POST请求
  • 认证方式使用token
  • 请求体使用json编码
  • 签名为md5，放到请求头中

直播房间列表http响应

  • 响应内容为json
  • resposne['data']['data']为base64编码，编码前的二进制是加密过的
  • resposne['data']['key']为base64编码，预计与解密相关

加解密算法分析

app对api的响应数据使用了典型的混合加密系统，使用RSA(非对称加密算法)加密对数据进行加密的AES(对称加密算法)的密钥，既保证了安全又保证效率。

RSA非对称加密

  1. 对resposne['data']['key']进行base64解码

  2. 搜索定位到RSA私钥解密处

  3. 加载RSA私钥，key_hex_str–> key_bytes –> private_key_obj

  4. 对resposne['data']['key']进行解密

AES对称加密

  1. 对resposne['data']['data']进行base64解码
  2. 使用前面的aes_key_plain_bytes对data进行解密
