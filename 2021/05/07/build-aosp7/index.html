编译aosp7.1.2

发表于 2021-05-07 | 分类于 note

环境信息

  • 本地环境: win10 git-bash
  • 编译服务器：腾讯云 香港地区 ubuntu18.04 16核32G
  • 工作目录：/data/aosp7.1.2r8 (挂载了一块500G的硬盘 mount /dev/vdb /data)
  • aosp版本: android-7.1.2_r8 N2G47O
  • 手机: pixel1(sailfish)

拉取源码

拉取源码有两种方式，通过官方工具repo、直接通过网盘下载

方案 占用磁盘大小 时间
从网盘下载 压缩包8G，解压后22G
使用repo拉取 168G 6小时

#方式1. 通过repo拉取源码

1.init git

2. install repo

3. repo init repostry && branch

4. repo sync branch

#方式2. 通过网盘下载源码包

这种方式仅下载了相关版本的源码，并且无git历史版本信息，所以会比repo拉取来得快，空间占用也会少很多。

  1. 网盘下载aosp712r8.7z 提取码：6666
  2. 从本地上传到服务器，可以使用scp
  3. 解压/data# 7z x aosp712r8.7z

Q&A

Q: SSL error when connecting to the Jack server. Try ‘jack-diagnose’
A: open-jdk8，默认禁用了TLSv1, TLSv1.1，需要编辑配置文件/etc/java-8-openjdk/security/java.security，去除jdk.tls.disabledAlgorithms中的TLSv1, TLSv1.1

Q: build/core/ninja.mk:148: recipe for target ‘ninja_wrapper’ failed
A: 内存不足或内存溢出了，export JACK_SERVER_VM_ARGUMENTS="-Dfile.encoding=UTF-8 -XX:+TieredCompilation -Xmx4096m"

Q: FAILED: /bin/bash -c "prebuilts/misc/linux-x86/flex/flex-2.5.39 -oout/host/linux-x86/obj/STATIC_LIBRARIES/libaidl-common_intermediates/aidl_language_l.cpp system/tools/aidl/aidl_language_l.ll" flex-2.5.39: loadlocale.c:130: _nl_intern_locale_data: Assertion cnt < (sizeof (_nl_value_type_LC_TIME) / sizeof (_nl_value_type_LC_TIME[0]))' failed.
A: export LANG=C
